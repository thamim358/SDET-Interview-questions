Explain the difference between var, let, and const with hoisting and scope. Advanced P1
Basic Answer

var works inside the whole function. let and const work only inside the nearest {} block. var is hoisted with undefined. let and const are hoisted but not ready to use (Temporal Dead Zone). const cannot be reassigned, but you can change object properties inside it. let can be reassigned. Rule: always use const first. Use let only if you need to reassign. Never use var in new code.

Detailed Answer

Scope (Where can you use it?): var works inside the whole function — even outside blocks like if or for. let and const work only inside one block {}. If you declare them inside if, you can't use them outside.

Hoisting (Moved to top?): All three are moved to the top of their scope. var is given the starting value undefined. let and const are moved to top but NOT given a starting value. If you try to use them before declaring, you get a ReferenceError. This is called the Temporal Dead Zone (TDZ).

Reassignment (Can you change it?): var and let can be changed to a new value. const cannot be reassigned — but if it's an object, you can still change its properties (like obj.name = 'new').

Global Declaration (Window object): var at the top level adds a property to window/globalThis. let and const do NOT.

// var — function scoped, hoisted with undefined console.log(x); // undefined (not ReferenceError) var x = 5; // let/const — block scoped, Temporal Dead Zone console.log(y); // ReferenceError: Cannot access before initialization let y = 10; // const — must be initialized at declaration const obj = { name: 'Test' }; obj.name = 'Updated'; // Allowed — object not frozen // obj = { name: 'New' }; // TypeError: Assignment to constant variable
💡 Interview Insight Interviewers ask this to check if you really understand hoisting and scope. Always mention the TDZ — it shows you know the details. For senior roles, give a real example: "We had a bug because var leaked outside a for loop block."
Real Interview Answer

"var works across the whole function and gets hoisted with undefined. let and const only work inside the block they're declared in. They are hoisted too but stuck in the Temporal Dead Zone until you reach them. const can't be reassigned, but object properties can still change. In my test code, I use const by default. I use let only when I need to reassign (like loop counters). I never use var."

⚠️ Common Mistake Many candidates say "let and const are not hoisted" — this is incorrect. They ARE hoisted, but not initialized. The TDZ is the period between entering scope and the actual declaration. Always clarify this distinction.

What is a closure? Give real-world examples of closures in test automation. Advanced P1
Basic Answer

A closure is a function that remembers variables from where it was created, even after that outer function has finished running. This lets you keep data private and save state. In test automation, closures are used for: keeping retry attempt counts, creating custom fixture setup/teardown, building reusable page object factories, and creating custom assertion matchers.

Detailed Answer

How it works: When you create a function inside another function, the inner function keeps a reference to the outer function's variables. JavaScript keeps those variables alive as long as any inner function still needs them.

Key point: Each time you create a closure, it gets its own copy of the outer variables. But if multiple closures are created in the same scope, they share the same variables — this is why the "loop with var" bug happens.

// Classic closure — retry with state function createRetryRunner(maxRetries, baseDelay) { let attemptCount = 0; // Private state — only accessible via closure return async function run(fn) { while (attemptCount < maxRetries) { try { return await fn(); } catch (error) { attemptCount++; if (attemptCount >= maxRetries) throw error; await new Promise(r => setTimeout(r, baseDelay * attemptCount)); } } }; } // Usage in Playwright test const retryClick = createRetryRunner(3, 500); await retryClick(() => page.locator('#submit').click()); // attemptCount is preserved between calls
📖 Memory Consideration Closures can cause memory leaks if they hold onto large objects. In test automation this is rarely a problem since tests are short-lived. But be careful not to capture big DOM elements or large API responses in closures.
Real Interview Answer

"A closure is a function that remembers the variables from where it was defined, even after that scope has exited. In my automation work, I use closures extensively — for example, creating a retry function that tracks attempt counts privately, building custom Playwright fixtures that capture page instances with their test context, or implementing data generators that maintain internal counters. The key benefit is encapsulation without needing classes."

How does the event loop work? Explain call stack, callback queue, microtask queue, and macrotask queue. Expert P1
Basic Answer

The event loop is how JavaScript handles async tasks even though it only has one thread. It keeps checking if the call stack is empty. If it is, it takes tasks from the microtask queue (high priority) or the macrotask queue (low priority) and runs them. This is why JavaScript can handle APIs, timers, and Promises without freezing the page.

Detailed Answer
Call Stack (LIFO — function calls)
↓
Web APIs (DOM, setTimeout, fetch, events)
↓                            ↓
Microtask Queue        Macrotask Queue
(Promise.then, await resumption)      (setTimeout, setInterval, I/O)
               ↓
        Event Loop (checks if stack is empty)
               ↓
        Back to Call Stack

Order of execution:

  1. Run all normal (sync) code first
  2. After each task, run ALL microtasks (Promise callbacks, queueMicrotask)
  3. Then run ONE macrotask (setTimeout, setInterval, I/O)
  4. Repeat — this is one "tick" of the event loop
// Event loop execution order prediction console.log(1); // Sync — runs first setTimeout(() => console.log(2), 0); // Macrotask — runs 4th Promise.resolve().then(() => console.log(3)); // Microtask — runs 3rd console.log(4); // Sync — runs 2nd // Output: 1, 4, 3, 2
⚠️ Interview Trap Many people say "setTimeout with 0ms runs right away." This is WRONG. Even with 0ms, it goes to the macrotask queue and runs AFTER all sync code and all microtasks. 0ms is the minimum delay, not the actual execution time.
Real Interview Answer

"The event loop is how JavaScript does async work with just one thread. The call stack runs functions one by one. When async code like setTimeout or a Promise is called, its callback goes to a queue — microtasks for Promises, macrotasks for timers. After each macrotask finishes, the event loop empties the ENTIRE microtask queue before moving to the next macrotask. This matters in tests — for example, when a Promise resolves before a waitForTimeout finishes in Playwright."

Knowing microtask/macrotask priority helped me fix it by awaiting the Promise directly instead of using a timer wait."

Explain the difference between == and ===. When would you use each? Intermediate P1
Basic Answer

== changes types before comparing (coercion). === compares both value AND type without changing anything. Always use ===. Only use == when you want to allow type changes — like checking null == undefined (which returns true) or checking if something is either null or undefined.

Detailed Answer

Coercion rules for ==:

  • Same type → compare directly (same as ===)
  • null == undefined → always true
  • String vs Number → string is converted to number
  • Boolean vs non-Boolean → boolean is converted to number
  • Object vs primitive → object is converted via .valueOf() or .toString()
console.log(5 == '5'); // true (coercion) console.log(5 === '5'); // false (different types) console.log(null == undefined); // true (special rule) console.log(null === undefined); // false console.log(0 == false); // true (false → 0) console.log('' == 0); // true console.log([] == false); // true ([] → '' → 0, false → 0)
Real Interview Answer

"I always use === in production code to avoid unexpected bugs from type coercion. The only time I use == is to check for both null and undefined at once — value == null catches both. In test assertions, I only use === or deep equality checks because type coercion can hide real bugs."

Follow-Up Questions

Explain how this binding works — default, implicit, explicit, arrow function, and new binding. Advanced P1
Basic Answer

this depends on HOW you call a function, not where you write it. Default: this is the global object (window) in normal mode, undefined in strict mode. Implicit: this is the object before the dot (like obj.foo() → this = obj). Explicit: use call(), apply(), or bind() to set this yourself. Arrow function: this comes from the surrounding code where the arrow function was written. new: this is the new object being created.

"this follows four rules in order of priority: new, then explicit (call/apply/bind), then implicit (the object before the dot), then default (global). Arrow functions skip all these rules — they get this from the code around them. In my Playwright page objects, I use arrow functions for methods so this doesn't get lost when I pass methods as callbacks."

What is the difference between call(), apply(), and bind()? Implement a polyfill for bind(). Advanced P1
Basic Answer

call() runs a function right away with a custom this — you pass arguments one by one. apply() does the same but takes arguments as an array. bind() does NOT run the function — it returns a NEW function with this permanently set. Use bind when you want to save the function for later with a fixed this.

Detailed Answer
// call — invoke with this and individual args greet.call(user, 'Hello', '!'); // apply — invoke with this and args array greet.apply(user, ['Hello', '!']); // bind — returns new function with bound this const boundGreet = greet.bind(user, 'Hello'); boundGreet('!'); // 'Hello' is already partially applied // Polyfill for bind Function.prototype.myBind = function(context, ...args) { const originalFn = this; return function(...innerArgs) { return originalFn.apply(context, [...args, ...innerArgs]); }; };
Real Interview Answer

"call and apply run a function right away with a custom this — call takes arguments one by one, apply takes an array. bind returns a new function with this locked in place, which is great for event handlers and borrowing methods. In Playwright, I use bind to keep this working correctly when passing page object methods as callbacks."

What is prototypal inheritance? How does the prototype chain work? Advanced P1
Basic Answer

Every JavaScript object has a hidden link to another object, called its prototype (__proto__ or Object.getPrototypeOf()). When you try to get a property from an object, JavaScript first checks that object. If it's not there, it checks the prototype, then that prototype's prototype, and so on until it finds it or reaches null. This lets objects share properties and methods without needing classes.

Real Interview Answer

"JavaScript uses prototypal inheritance — objects get features from other objects through the prototype chain. When you access a property, JavaScript walks up the chain until it finds it or hits null. This is different from class-based languages where you need a blueprint. In Playwright, understanding prototypes helps when extending built-in fixtures or creating reusable utilities."

What are Promises? Explain the states and how Promise chaining works. Intermediate P1
Basic Answer

A Promise is an object that represents a task that will finish later — either successfully or with an error. It has three states: pending (waiting), fulfilled (done successfully), and rejected (failed). You chain Promises with .then() — each .then() returns a new Promise. Errors travel down the chain until a .catch() catches them. Useful helpers: Promise.all() (wait for all), Promise.race() (first one wins), Promise.any() (first success wins), Promise.allSettled() (wait for all, even failures).

Detailed Answer
// Promise states and chaining const fetchUser = new Promise((resolve, reject) => { const user = { id: 1, name: 'Alice' }; if (user) resolve(user); else reject(new Error('User not found')); }); fetchUser .then(user => user.name) .then(name => console.log(name)) // 'Alice' .catch(err => console.error(err)) .finally(() => console.log('Done')); // Promise.all — fail-fast if any rejects const [users, posts] = await Promise.all([ fetch('/api/users'), fetch('/api/posts') ]); // Promise.allSettled — wait for all, no fail-fast const results = await Promise.allSettled([ fetch('/api/users'), fetch('/api/failing-endpoint') ]);
Real Interview Answer

"Promises are everywhere in Playwright — almost every action returns a Promise. I use Promise.all to make parallel API calls in test setup. I use Promise.allSettled when I want all results even if some fail. I use chaining for step-by-step operations like login → navigate → verify. Understanding how errors flow through the chain is key for good error handling."

Follow-Up Questions
How is a Promise microtask different from a setTimeout macrotask? What happens if you return a Promise inside a .then()? What is the Promise resolution procedure (the Promise Resolution Procedure)?

How does async/await work? How does error handling differ from Promise .then()/.catch()? Intermediate P1
Basic Answer

async/await is a cleaner way to write Promises. An async function always returns a Promise. await pauses the function until the Promise finishes — but it does NOT block the main thread. Use try/catch for error handling, which is easier to read than .catch() chains. Behind the scenes, await is just .then() with cleaner syntax.

Real Interview Answer

"I prefer async/await over raw Promises because it's easier to read — especially in Playwright tests where steps happen one after another. I wrap test code in try/catch for clean error handling. I use Promise.all inside async functions for parallel work. One trap: await inside a loop runs one at a time (slow!). Use Promise.all with map to run things in parallel."

What is hoisting? How does it differ for var, let, const, and function declarations? Intermediate P1
Basic Answer

Hoisting means JavaScript moves all declarations to the top of their scope during setup. Function declarations are fully hoisted — you can call them before writing them. var is hoisted and starts as undefined. let and const are hoisted but NOT initialized (Temporal Dead Zone). Function expressions and arrow functions follow the hoisting rules of the variable (var/let/const) they're assigned to.

Real Interview Answer

"Hoisting means declarations are processed before the code runs. For function declarations, you can call them before they appear in the file — great for putting helper functions at the bottom. For var, the declaration is hoisted but the value assignment stays in place, so it's undefined until that line. let and const are hoisted too, but using them before the declaration line throws a ReferenceError."

Explain event delegation, event bubbling, and event capturing in the DOM.IntermediateP1
Basic Answer

When an event happens on an element, it goes through three phases: (1) Capturing — from window down to the target. (2) Target — the element itself. (3) Bubbling — from the target back up to window. Event delegation uses bubbling — instead of adding a listener to each child, you add ONE listener to the parent and check which child triggered it. This saves memory and works for elements added after page load.

Real Interview Answer

"Event delegation is important for dynamic elements — common in modern single-page apps. Instead of adding listeners to every button, I add one to the parent and check event.target to find which child was clicked. This connects to Playwright's auto-waiting: understanding event flow helps debug timing issues when elements load dynamically."

What is the difference between map(), filter(), and reduce()? Provide real use cases.IntermediateP1
Basic Answer

map() changes every element and returns a new array (same length). filter() picks elements that pass a test and returns a new array (shorter or same). reduce() combines all elements into any single value (number, object, array). None of them change the original array. In testing: use map to get text from page elements, filter to remove invalid test cases, and reduce to add up test results.

Detailed Answer
// Extract prices from page elements const prices = await page.locator('.price').allTextContents(); const priceNumbers = prices .map(p => parseFloat(p.replace('$', ''))) .filter(p => p > 0); // Remove free items const totalPrice = priceNumbers.reduce((sum, p) => sum + p, 0);

How do you deep clone an object in JavaScript? What are the limitations of JSON.parse(JSON.stringify(obj))?IntermediateP1
Basic Answer

JSON.parse(JSON.stringify(obj)) is the easiest way to deep-clone an object, but it has big limitations: it drops functions, undefined, Symbols, BigInt, circular references, and converts Date to string, and loses Map, Set, and RegExp. For proper deep cloning, use structuredClone() (Node 17+ or modern browsers) or Lodash's _.cloneDeep().

Real Interview Answer

"For simple API response data, JSON.parse(JSON.stringify()) works fine. But for objects with Dates, Maps, or circular references — common in complex test configs — I use structuredClone() which handles all these. Deep cloning is important in test automation so each test gets its own copy of test data without sharing."

What are arrow functions? How do they differ from regular functions regarding this, arguments, and new?IntermediateP1
Basic Answer

Arrow functions are a shorter way to write functions. They differ from regular functions in three ways: (1) No own this — they take this from the surrounding code. (2) No arguments object — use ...args instead. (3) Can't be used with new as constructors. They also can't be generators (no yield). Use arrow functions for callbacks and methods where you don't want this to change.

What is the nullish coalescing operator (??) and how does it differ from ||?IntermediateP1
Basic Answer

?? (nullish coalescing) gives the right value ONLY when the left value is null or undefined. || (OR) gives the right value for ALL falsy values — false, 0, '', null, undefined, NaN. Use ?? when you want to keep valid values like 0 or empty string as real values. This matters for test config — config.timeout ?? 30000 keeps 0 as a real value, while config.timeout || 30000 would replace 0 with 30000 (a bug!).

const timeout = config.timeout ?? 30000; // Uses 0 if explicitly set to 0 const timeout2 = config.timeout || 30000; // Replaces 0 with 30000 — bug!

What is the spread operator vs rest operator? Give practical test automation examples.IntermediateP1
Basic Answer

Spread (...) spreads items out — like unpacking an array into separate values. Rest (...) collects items together — like grouping function arguments into an array. Use spread to copy arrays/objects, merge them, or pass array items as function arguments. Use rest for functions with any number of arguments or destructuring. In Playwright, spread is great for merging default config with overrides: { ...defaultConfig, ...overrides }.

How does Object.freeze() differ from Object.seal() and Object.preventExtensions()?AdvancedP1
Basic Answer

Object.preventExtensions() — stops you from adding new properties. Object.seal() — stops add + delete. Object.freeze() — stops add + delete + change values. All three are shallow — nested objects can still be changed. Use freeze for configuration objects that should never change in your test framework.

Implement debounce and throttle functions. When would you use each in test automation?AdvancedP1
Basic Answer

Debounce: waits until the user stops calling the function for X time, then runs it once. Like waiting for someone to stop typing before searching. Throttle: runs the function at most once every X time, no matter how many times it's called. Like limiting API calls to once per second. In testing: debounce for search input tests, throttle for scroll event or rate limit tests.

function debounce(fn, delay) { let timer; return function(...args) { clearTimeout(timer); timer = setTimeout(() => fn.apply(this, args), delay); }; } function throttle(fn, interval) { let lastCall = 0; return function(...args) { const now = Date.now(); if (now - lastCall >= interval) { lastCall = now; fn.apply(this, args); } }; }

What is the difference between Map and WeakMap? What about Set and WeakSet?AdvancedP1
Basic Answer

Map vs WeakMap: Map keeps references strong (object stays in memory). WeakMap keys must be objects, and if nothing else references that object, it gets cleaned up (garbage collected). WeakMap can't be looped over and has no .size. Use WeakMap to attach data to DOM elements without causing memory leaks. Same for Set vs WeakSet. In Playwright, WeakMap helps cache element references safely.

What is NaN? How do you reliably check if a value is NaN?IntermediateP1
Basic Answer

NaN stands for "Not-a-Number." It's a special value that represents an invalid number. NaN is the only value in JavaScript that is NOT equal to itself (NaN !== NaN is true). To check for NaN, use Number.isNaN(). DON'T use the global isNaN() — it converts the value to a number first, so isNaN('hello') returns true (wrong!), while Number.isNaN('hello') returns false (correct).

isNaN('hello'); // true — coerces 'hello' → NaN Number.isNaN('hello'); // false — no coercion Number.isNaN(NaN); // true

What are the primitive data types in JavaScript? How do they differ from reference types?BeginnerP2
Basic Answer

Primitive types (7): string, number, boolean, null, undefined, symbol, bigint. They are stored directly as values — like putting a number in a box. Reference types: objects, arrays, functions. They are stored as a reference (a pointer) to where the data lives in memory. When you compare objects, JavaScript checks if they point to the SAME memory location, not if they have the same content. This is why two identical objects aren't equal — you need deep comparison (toEqual vs toBe).

Real Interview Answer

"JavaScript has 7 primitive types and reference types. Primitives store the actual value — let b = a (where a is a number) creates a copy. Objects and arrays store a reference — assigning copies the address, not the data. This matters in test assertions: toBe checks reference (same object?), toEqual checks values (same content?)."

What are truthy and falsy values in JavaScript? How do they affect conditional logic in tests?BeginnerP2
Basic Answer

There are only 6 falsy values in JavaScript: false, 0, '' (empty string), null, undefined, NaN. EVERYTHING else is truthy — including '0', [], {}, and 'false'. This matters in tests: if (element) is true for ANY object, even an empty array. Always be explicit about what you're checking.

Similarly, if (errorCode) fails when errorCode is 0, which might be a valid error code. I always use explicit comparisons in test assertions."

How does destructuring work for arrays, objects, and nested structures?IntermediateP2
Basic Answer

Destructuring unpacks values from arrays/objects into variables in one line. Array destructuring uses position: const [first, second] = arr. Object destructuring uses property names: const { name, age } = obj. You can set defaults, rename variables, and handle nested data. Great for extracting fields from API responses.

// API response destructuring const { id, name, address: { city, country } } = await response.json(); expect(id).toBe(1); expect(city).toBe('New York');
31
What is currying? How is it useful in test configuration?AdvancedP2
Basic Answer

Currying turns a function that takes multiple arguments into a chain of functions that each take one argument. f(a, b, c) becomes f(a)(b)(c). Use this to create reusable test helpers: const createUser = (role) => (overrides) => ({ ...baseConfig, role, ...overrides }).

32
How do you remove duplicates from an array? Compare Set vs filter approaches.IntermediateP2
Basic Answer

For simple values: [...new Set(arr)] — easiest and fast (O(n)). For objects: Use filter with findIndex — slower (O(n²)) but works. For large object arrays: Use a Map with a unique key for best performance.

33
What is memoization? Implement a memoized function.AdvancedP2
Basic Answer

Memoization stores (caches) the result of a function call so if you call it again with the same arguments, it returns the cached result instead of recalculating. Useful for expensive operations. The cache is usually a Map or object keyed by the function's arguments.

function memoize(fn) { const cache = new Map(); return function(...args) { const key = JSON.stringify(args); if (cache.has(key)) return cache.get(key); const result = fn.apply(this, args); cache.set(key, result); return result; }; }
34
What is the difference between splice() and slice()?BeginnerP2
Basic Answer

slice(start, end) returns a new array (does NOT change the original). splice(start, deleteCount, ...items) changes the original array (adds/removes items). Remember: slice = copy (safe), splice = change (mutates).

35
How do you flatten a nested array? Implement both recursive and flat() approaches.IntermediateP2
Basic Answer

Use arr.flat(depth) to flatten nested arrays. For unlimited depth: arr.flat(Infinity). A recursive approach uses reduce and concat. In tests, flattening helps process multi-level API response data.

36
What are generators and iterators? When would you use them?AdvancedP2
Basic Answer

Generators (function*) are functions that can pause and resume. Use yield to return values one at a time. They return an iterator with .next(). Great for generating large test datasets lazily (only when needed), infinite sequences, or custom iteration patterns.

37
How does short-circuit evaluation work with && and ||?BeginnerP2
Basic Answer

&& returns the first falsy value, or the last value if all are truthy. || returns the first truthy value, or the last value if all are falsy. Short-circuit means JavaScript stops evaluating as soon as it knows the result. Used for: isLoggedIn && redirect() (run if true) or defaults: const name = user.name || 'Guest'.

38
How does forEach() differ from map()? When should you avoid forEach()?BeginnerP2
Basic Answer

forEach() runs a function on each element but returns nothing. map() runs a function on each element and returns a NEW array with the results. Use forEach for side-effects (logging, clicking elements). Use map for transformations (extracting data). In tests: prefer map for data extraction, forEach for one-by-one element interaction.

39
Explain how reduce() can be used for grouping, counting, and flattening.IntermediateP2
Basic Answer

reduce() is the most flexible array method — it can do almost anything. Grouping: group items by category. Counting: count how many of each item. Flattening: turn a 2D array into a 1D array. Think of reduce as: start with a starting value, then combine each element into it one by one.

40
What is an IIFE? Why would you use one?IntermediateP2
Basic Answer

IIFE stands for Immediately Invoked Function Expression. It's a function that runs right away when it's defined: (function(){ ... })(). It creates a private scope so variables don't leak to the global scope. Used in module patterns before ES6 modules came along. Still useful for keeping code isolated.

41
What are template literals and tagged templates? Give a Playwright locator example.IntermediateP2
Basic Answer

Template literals use backticks (`` ` ``) and let you insert variables with ${expr}. Tagged templates let you process template literals with a function. In Playwright, they're great for dynamic locators: page.locator(`[data-testid="user-${userId}"]`).

42
What is the difference between local storage, session storage, and cookies in browser storage?IntermediateP2
Basic Answer

Local Storage: stays until you clear it, shared across tabs. Session Storage: lives only in the current tab, gone when tab closes. Cookies: sent with every HTTP request, max 4KB, can expire. In Playwright, use page.evaluate(() => localStorage.getItem('token')) to access storage, and storageState to save auth across tests.

43
How does the typeof operator work? What is typeof null?BeginnerP2
Basic Answer

typeof returns a string saying what type something is. typeof null returns 'object' — this is a famous bug from JavaScript's early days. Use value === null to check for null. Other quirks: typeof NaN returns 'number', typeof [] returns 'object' (use Array.isArray() instead).

44
How do you implement a simple Promise from scratch?ExpertP2
Basic Answer

A basic Promise needs: a state machine (pending → fulfilled/rejected), an executor function, resolve/reject callbacks, and a .then() method for chaining. The executor runs right away. .then() queues callbacks that run when the state changes (or immediately if already done). For proper async behavior, use queueMicrotask.

45
How does JavaScript handle type coercion? Give examples of implicit vs explicit coercion.IntermediateP2
Basic Answer

Implicit coercion (JavaScript changes types automatically): '5' - 3 → 2 (string becomes number), '5' + 3 → '53' (number becomes string), !0 → true. Explicit coercion (you do it on purpose): Number('5'), String(5), Boolean(0). The + operator prefers to make strings if either side is a string.

46
What is the difference between for...in and for...of?BeginnerP2
Basic Answer

for...in loops over property NAMES (keys). Use for objects. for...of loops over property VALUES. Use for arrays, strings, Maps, Sets. for...in on arrays can include prototype properties — avoid it for arrays.

47
How does Object.create() work? Compare with constructor functions and class syntax.AdvancedP2
Basic Answer

Object.create(proto) creates a new object with a given prototype — pure prototypal inheritance without constructors. Constructor functions (new Fn()) and classes (class) are just easier ways to use prototypes. Object.create(null) creates an object with NO prototype — useful for dictionaries that shouldn't inherit anything.

48
What is the difference between Array.isArray() and instanceof Array?IntermediateP2
Basic Answer

Use Array.isArray() to check if something is an array — it's the reliable way. instanceof Array can fail if the array comes from a different frame or context (different Array constructor). Array.isArray() works everywhere.

What is strict mode ("use strict") and what does it prevent?IntermediateP3
Basic Answer

Strict mode ("use strict") makes JavaScript stricter and catches more errors. It prevents: using variables without declaring them, deleting things that can't be deleted, duplicate function parameter names, the with statement, this being the global object in functions, and setting properties on primitive values. Instead of silently failing, JavaScript throws errors.

What are Symbols and where are they useful?AdvancedP3
Basic Answer

Symbols are unique, unchangeable values used as object property keys to avoid name conflicts. Symbol.for('key') creates shared global symbols. Used for custom iterators, metadata, and framework internals. Not needed in daily test automation, but useful for designing utility libraries.

Explain the Proxy API and give a test automation use case.ExpertP3
Basic Answer

Proxy wraps an object and lets you intercept operations (get, set, has, delete, etc.). Use cases: a logging proxy that records every property access on a test config object, or a validation proxy that checks required properties are set before tests run.

52
How does Intl work for localization testing?AdvancedP3
Basic Answer

The Intl API formats dates, numbers, and currencies for different countries/languages. Useful for localization testing — checking that your app shows the right format for each region. Example: new Intl.DateTimeFormat('de-DE').format(date) vs 'en-US'.

53
What is the Reflect API? How does it relate to Proxy?ExpertP3
Basic Answer

Reflect is a global object with methods that match Proxy traps (Reflect.get(), Reflect.set(), etc.). It provides the default behavior for Proxy traps. Use Reflect inside Proxy traps to forward operations to the original object.

54
How do you check if an object has a property (own vs inherited)?IntermediateP3
Basic Answer

obj.hasOwnProperty('key') or Object.hasOwn(obj, 'key') — checks ONLY the object's own properties (not inherited). 'key' in obj — checks own AND inherited properties. Don't use obj.key !== undefined — the property could exist but be set to undefined.

55
What are tagged templates and how would you use them for building safe SQL queries?AdvancedP3
Basic Answer

Tagged templates let you process template literals with a function that receives the string parts and the interpolated values separately. Useful for: safely escaping user input in SQL queries (like parameterized queries), building safe HTML, or creating custom mini-languages.

56
What is the difference between requestAnimationFrame and setTimeout?AdvancedP3
Basic Answer

requestAnimationFrame schedules code to run before the next browser screen update (~60fps). setTimeout runs after a delay regardless of rendering. requestAnimationFrame pauses when the tab is inactive (saves battery/resources). Useful for testing animations and visual changes.

57
What are Web Workers and how do they relate to the event loop?ExpertP3
Basic Answer

Web Workers run JavaScript in a background thread — they have their own event loop and call stack. They talk to the main thread using postMessage. They can't access the DOM. Useful for offloading heavy calculations (like generating test data) without freezing the page.

58
How does Error.captureStackTrace() work and when is it useful?ExpertP3
Basic Answer

Error.captureStackTrace(target, constructorOpt) is a V8 method that creates a .stack property on an object. It captures the call stack at the moment it's called. Useful in custom assertion libraries to make error stack traces point to the actual test line, not inside the assertion library.

📋 JavaScript — One-Page Revision Cheat Sheet

Scope & Hoisting

var → whole function, hoisted as undefined

let/const → inside one block (TDZ)

Function declarations → fully hoisted (can call before writing)

Closures

Function remembers its outer variables

Used for: private data, retry logic, factory functions

Risk: memory leaks if holding big objects

Event Loop

1. Run all sync code first

2. Run ALL microtasks (Promises)

3. Run ONE macrotask (setTimeout)

4. Repeat

this Priority

new > explicit (call/apply/bind) > implicit (obj.method) > default (global)

Arrow functions: inherit from surrounding code

Promise Helpers

Promise.all → fails on first error

Promise.allSettled → waits for all, even failures

Promise.race → first one to finish wins

Promise.any → first success wins

Object Methods

freeze → no add/delete/change

seal → no add/delete, can change values

preventExtensions → no add

All are shallow (nested objects still changeable)

Array Methods

map → transform each item (returns new array)

filter → keep items that pass test (returns new array)

reduce → combine all into one value

find → first match

some/every → true/false check

Cloning

Shallow: spread (...obj), Object.assign()

Deep: structuredClone(), JSON.parse/stringify (limited)

Watch out: functions, Dates, Maps, Sets, circular refs
