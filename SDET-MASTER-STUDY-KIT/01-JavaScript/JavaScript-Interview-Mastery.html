Explain the difference between var, let, and const with hoisting and scope. Advanced P1
Basic Answer

var is function-scoped and hoisted with undefined initialization. let and const are block-scoped and hoisted but not initialized (Temporal Dead Zone). const cannot be reassigned, but its object properties can be mutated. let allows reassignment. Always prefer const by default, use let when reassignment is needed, and never use var in modern code.

Detailed Answer

Scope: var is function-scoped — visible throughout the function regardless of block boundaries. let and const are block-scoped — confined to the nearest {} block.

Hoisting: All three are hoisted to the top of their scope. var is initialized as undefined automatically. let and const are hoisted but remain uninitialized — accessing them before declaration throws a ReferenceError due to the Temporal Dead Zone (TDZ).

Reassignment: var and let can be reassigned. const cannot be reassigned, but it doesn't make the value immutable — object properties can still be modified.

Global Declaration: var at global scope creates a property on window/globalThis. let and const do not.

// var — function scoped, hoisted with undefined console.log(x); // undefined (not ReferenceError) var x = 5; // let/const — block scoped, Temporal Dead Zone console.log(y); // ReferenceError: Cannot access before initialization let y = 10; // const — must be initialized at declaration const obj = { name: 'Test' }; obj.name = 'Updated'; // Allowed — object not frozen // obj = { name: 'New' }; // TypeError: Assignment to constant variable
💡 Interview Insight Interviewers ask this to test your understanding of hoisting mechanics and scope. Always mention the TDZ explicitly — it signals deep knowledge. Senior candidates should connect this to real issues: "We once had a bug caused by var leaking out of a for loop block."
Real Interview Answer

"var is function-scoped and gets hoisted with an undefined initialization. let and const are block-scoped — they're hoisted too but remain in the Temporal Dead Zone until the declaration is reached. The key difference between let and const is reassignment: const can't be reassigned, though object properties can still change. In modern test automation code, I use const by default, let when I need to reassign (like loop counters), and never var."

What is a closure? Give real-world examples of closures in test automation. Advanced P1
Basic Answer

A closure is a function that retains access to variables from its outer (enclosing) scope even after the outer function has returned. This allows data encapsulation, private variables, and state preservation. In test automation, closures are used for fixture setup/teardown, retry logic with state, custom matchers, and page object factories.

Detailed Answer

How it works: When a function is defined inside another function, the inner function maintains a reference to the outer function's variables through its [[Scope]] property. The JavaScript engine keeps these variables alive in memory as long as any closure references them.

Key properties: Each closure captures its own copy of the outer variables. Multiple closures created in the same scope share the same variable reference — this is the classic "loop with var" pitfall.

// Classic closure — retry with state function createRetryRunner(maxRetries, baseDelay) { let attemptCount = 0; // Private state — only accessible via closure return async function run(fn) { while (attemptCount < maxRetries) { try { return await fn(); } catch (error) { attemptCount++; if (attemptCount >= maxRetries) throw error; await new Promise(r => setTimeout(r, baseDelay * attemptCount)); } } }; } // Usage in Playwright test const retryClick = createRetryRunner(3, 500); await retryClick(() => page.locator('#submit').click()); // attemptCount is preserved between calls
📖 Memory Consideration Closures can cause memory leaks if the outer scope holds large objects. In test automation, this is rarely an issue since test processes are short-lived, but be mindful of closures capturing entire DOM elements or large response payloads.
Real Interview Answer

"A closure is a function that remembers the variables from where it was defined, even after that scope has exited. In my automation work, I use closures extensively — for example, creating a retry function that tracks attempt counts privately, building custom Playwright fixtures that capture page instances with their test context, or implementing data generators that maintain internal counters. The key benefit is encapsulation without needing classes."

Follow-Up Questions

How does the event loop work? Explain call stack, callback queue, microtask queue, and macrotask queue. Expert P1
Basic Answer

The event loop is JavaScript's mechanism for handling asynchronous operations despite being single-threaded. It continuously checks if the call stack is empty, and if so, pushes pending tasks from the microtask queue (higher priority) and macrotask queue (lower priority) onto the stack. This enables non-blocking I/O, timers, and Promise-based operations.

Detailed Answer
Call Stack (LIFO — function calls)
↓
Web APIs (DOM, setTimeout, fetch, events)
↓                            ↓
Microtask Queue        Macrotask Queue
(Promise.then, await resumption)      (setTimeout, setInterval, I/O)
               ↓
        Event Loop (checks if stack is empty)
               ↓
        Back to Call Stack

Execution Order:

  1. Execute all synchronous code on the call stack
  2. After each macrotask, drain the entire microtask queue (all Promise callbacks, queueMicrotask)
  3. Take one macrotask from the macrotask queue (setTimeout, setInterval, I/O)
  4. Repeat — this is one "tick" of the event loop
// Event loop execution order prediction console.log(1); // Sync — runs first setTimeout(() => console.log(2), 0); // Macrotask — runs 4th Promise.resolve().then(() => console.log(3)); // Microtask — runs 3rd console.log(4); // Sync — runs 2nd // Output: 1, 4, 3, 2
⚠️ Interview Trap Many candidates say "setTimeout with 0ms delay runs immediately." This is wrong. Even with 0ms delay, the callback is placed in the macrotask queue and runs after all synchronous code AND all microtasks have completed. Always clarify that 0ms is a minimum, not a guaranteed execution time.
Real Interview Answer

"The event loop is what makes JavaScript's single-threaded concurrency model work. The call stack executes functions synchronously. When an async operation like setTimeout or a Promise is encountered, its callback goes to the appropriate queue — microtasks for Promises and mutation observers, macrotasks for timers and I/O. After each macrotask completes, the event loop drains the ENTIRE microtask queue before processing the next macrotask. This is critical for understanding async test execution — especially when debugging why a Promise resolved before a waitForTimeout in Playwright."

Explain the difference between == and ===. When would you use each? Intermediate P1
Basic Answer

== (abstract equality) performs type coercion before comparing values. === (strict equality) compares both value AND type without coercion. Always use === by default. Only use == when you explicitly want type coercion — e.g., comparing null == undefined (returns true) or checking if a value is either null or undefined.

Detailed Answer

Coercion rules for ==:

  • Same type → compare directly (same as ===)
  • null == undefined → always true
  • String vs Number → string is converted to number
  • Boolean vs non-Boolean → boolean is converted to number
  • Object vs primitive → object is converted via .valueOf() or .toString()
console.log(5 == '5'); // true (coercion) console.log(5 === '5'); // false (different types) console.log(null == undefined); // true (special rule) console.log(null === undefined); // false console.log(0 == false); // true (false → 0) console.log('' == 0); // true console.log([] == false); // true ([] → '' → 0, false → 0)
Real Interview Answer

"I use === by default in all production code to avoid unexpected coercion bugs. The only place I intentionally use == is when checking for both null and undefined simultaneously — value == null catches both. In test assertions, I exclusively use === or deep equality checks because type coercion can mask real bugs."

Explain how this binding works — default, implicit, explicit, arrow function, and new binding. Advanced P1
Basic Answer

this is determined by how a function is called, not where it's defined. Default binding: this is global (window/globalThis) in non-strict mode, undefined in strict mode. Implicit binding: this is the object the method is called on. Explicit binding: call(), apply(), bind() set this explicitly. Arrow function: this is lexically bound — captured from enclosing scope. new binding: this refers to the newly created instance.

💡 Interview Insight The most common test automation issue with this is in event handlers and page object methods. When passing a method as a callback, this is lost. The fix is either bind(this) or using an arrow function.
Real Interview Answer

"this binding has four main rules in order of precedence: new binding, explicit binding via call/apply/bind, implicit binding from the object before the dot, and default binding to the global object. Arrow functions override all of these with lexical binding — they capture this from the enclosing scope. In my Playwright page objects, I use arrow functions for methods to avoid losing this when passing methods as callbacks to assertions or event listeners."

What is the difference between call(), apply(), and bind()? Implement a polyfill for bind(). Advanced P1
Basic Answer

call() invokes a function with a given this value and arguments passed individually. apply() is the same but arguments are passed as an array. bind() returns a NEW function with the this value permanently bound, without invoking it. All three are used for explicit this binding.

Detailed Answer
// call — invoke with this and individual args greet.call(user, 'Hello', '!'); // apply — invoke with this and args array greet.apply(user, ['Hello', '!']); // bind — returns new function with bound this const boundGreet = greet.bind(user, 'Hello'); boundGreet('!'); // 'Hello' is already partially applied // Polyfill for bind Function.prototype.myBind = function(context, ...args) { const originalFn = this; return function(...innerArgs) { return originalFn.apply(context, [...args, ...innerArgs]); }; };
Real Interview Answer

"call and apply are for immediate invocation with a custom this — call spreads arguments individually, apply takes an array. bind returns a new function with this permanently bound, which is useful for method borrowing and event handlers. In Playwright, I use bind when passing page object methods as callbacks to keep this context intact."

What is prototypal inheritance? How does the prototype chain work? Advanced P1
Basic Answer

Every JavaScript object has an internal [[Prototype]] (accessible via __proto__ or Object.getPrototypeOf()). When a property is accessed on an object, JavaScript first checks the object itself, then walks up the prototype chain until it finds the property or reaches null. This enables property sharing and memory-efficient method inheritance without classes.

Real Interview Answer

"JavaScript uses prototypal inheritance — objects inherit directly from other objects via the prototype chain. When you access a property, the engine looks up the chain until it finds it or reaches null. This is different from classical inheritance where classes define blueprints. In Playwright, understanding this helps when extending built-in fixtures or creating reusable test utilities."

What are Promises? Explain the states and how Promise chaining works. Intermediate P1
Basic Answer

A Promise is an object representing the eventual completion or failure of an asynchronous operation. It has three states: pending (initial state), fulfilled (resolved successfully), and rejected (failed). Promise chaining allows sequential async operations where each .then() returns a new Promise. Errors propagate through the chain until caught by a .catch(). Promise static methods include Promise.all(), Promise.race(), Promise.any(), and Promise.allSettled().

Detailed Answer
// Promise states and chaining const fetchUser = new Promise((resolve, reject) => { const user = { id: 1, name: 'Alice' }; if (user) resolve(user); else reject(new Error('User not found')); }); fetchUser .then(user => user.name) .then(name => console.log(name)) // 'Alice' .catch(err => console.error(err)) .finally(() => console.log('Done')); // Promise.all — fail-fast if any rejects const [users, posts] = await Promise.all([ fetch('/api/users'), fetch('/api/posts') ]); // Promise.allSettled — wait for all, no fail-fast const results = await Promise.allSettled([ fetch('/api/users'), fetch('/api/failing-endpoint') ]);
Real Interview Answer

"Promises are fundamental to Playwright — almost every operation returns a Promise. I use Promise.all for parallel API calls in test setup, Promise.allSettled when I want to collect results without failing fast, and chaining for sequential operations like login → navigate → verify. Understanding that rejections propagate through the chain is critical for writing robust error handling."

Follow-Up Questions
How is a Promise microtask different from a setTimeout macrotask? What happens if you return a Promise inside a .then()? What is the Promise resolution procedure (the Promise Resolution Procedure)?

How does async/await work? How does error handling differ from Promise .then()/.catch()? Intermediate P1
Basic Answer

async/await is syntactic sugar over Promises. An async function always returns a Promise. await pauses execution of the function until the awaited Promise settles, without blocking the main thread. Error handling uses try/catch blocks, which is more readable than .catch() chaining. Under the hood, await is equivalent to calling .then() on the Promise and scheduling the rest of the function as a microtask continuation.

Real Interview Answer

"I prefer async/await over raw Promises for readability — especially in Playwright tests where operations are inherently sequential. I wrap test logic in try/catch for clean error handling and use Promise.all within the same function for parallel operations. One trap: forgetting that await in a loop executes sequentially — use Promise.all with map for parallelism."

What is hoisting? How does it differ for var, let, const, and function declarations? Intermediate P1
Basic Answer

Hoisting is JavaScript's behavior of moving declarations to the top of their scope during compilation. Function declarations are fully hoisted (can be called before definition). var is hoisted and initialized as undefined. let and const are hoisted but uninitialized (Temporal Dead Zone). Function expressions and arrow functions assigned to variables follow the variable's hoisting behavior.

Real Interview Answer

"Hoisting means declarations are processed before execution. For function declarations, this means you can call them before they appear in the code — useful for organizing helper functions at the bottom of a file. For var, the declaration is hoisted but the assignment stays in place, so the value is undefined until the assignment line. let and const are hoisted too, but accessing them before the declaration throws a ReferenceError."

Event delegation leverages bubbling by attaching a single listener to a parent element to handle events from multiple children. This is memory-efficient and handles dynamically added elements.

This relates to Playwright's auto-waiting: understanding the event flow helps debug timing issues where elements are added dynamically after initial page load."

What is the difference between map(), filter(), and reduce()? Provide real use cases.IntermediateP1
Basic Answer

map() transforms every element and returns a new array of the same length. filter() returns a new array with elements that pass a test. reduce() accumulates values into any data type (number, array, object). All three are immutable — they don't modify the original array. In test automation, use map to extract text from elements, filter to exclude invalid test cases, and reduce to aggregate test results.

Detailed Answer
// Extract prices from page elements const prices = await page.locator('.price').allTextContents(); const priceNumbers = prices .map(p => parseFloat(p.replace('$', ''))) .filter(p => p > 0); // Remove free items const totalPrice = priceNumbers.reduce((sum, p) => sum + p, 0);

How do you deep clone an object in JavaScript? What are the limitations of JSON.parse(JSON.stringify(obj))?IntermediateP1
Basic Answer

JSON.parse(JSON.stringify(obj)) is the simplest deep clone method but has significant limitations: it drops functions, undefined values, Symbols, BigInt, circular references, and special objects like Date (converts to string), Map, Set, RegExp. For robust deep cloning, use structuredClone() (modern browsers/Node 17+) or a library like Lodash's _.cloneDeep().

Real Interview Answer

"For simple API response data, JSON.parse(JSON.stringify()) works fine. But when cloning objects with Dates, Maps, or circular references — common in complex test configuration objects — I use structuredClone() which handles these natively. In test automation, deep cloning is essential for creating independent copies of test data configurations."

What are arrow functions? How do they differ from regular functions regarding this, arguments, and new?IntermediateP1
Basic Answer

Arrow functions are a shorter syntax for function expressions. They differ from regular functions in three key ways: (1) No own this — they inherit this from the enclosing lexical scope. (2) No arguments object — use rest parameters instead. (3) Cannot be used as constructors with new. They also cannot be used as generators (no yield). Arrow functions are ideal for callbacks and methods that shouldn't rebind this.

What is the nullish coalescing operator (??) and how does it differ from ||?IntermediateP1
Basic Answer

?? returns the right-hand operand only when the left-hand operand is null or undefined. || returns the right-hand operand for ALL falsy values (false, 0, '', null, undefined, NaN). Use ?? when you want to treat only null/undefined as missing, but preserve valid falsy values like 0 or empty strings. This is critical for test configuration defaults.

const timeout = config.timeout ?? 30000; // Uses 0 if explicitly set to 0 const timeout2 = config.timeout || 30000; // Replaces 0 with 30000 — bug!

How does Object.freeze() differ from Object.seal() and Object.preventExtensions()?AdvancedP1
Basic Answer

Object.preventExtensions() prevents adding new properties. Object.seal() does that plus makes all properties non-configurable (can't delete). Object.freeze() does all of the above plus makes all properties non-writable (can't modify values). All are shallow — nested objects remain mutable. Use freeze for immutable configuration objects in test frameworks.

Implement debounce and throttle functions. When would you use each in test automation?AdvancedP1
Basic Answer

Debounce delays execution until a specified time has passed since the last invocation. Throttle ensures execution happens at most once per specified interval. In test automation: debounce is useful for search input testing (wait for user to stop typing before searching), throttle is useful for scroll event testing or API rate limiting validation.

function debounce(fn, delay) { let timer; return function(...args) { clearTimeout(timer); timer = setTimeout(() => fn.apply(this, args), delay); }; } function throttle(fn, interval) { let lastCall = 0; return function(...args) { const now = Date.now(); if (now - lastCall >= interval) { lastCall = now; fn.apply(this, args); } }; }

What is the difference between Map and WeakMap? What about Set and WeakSet?AdvancedP1
Basic Answer

WeakMap keys must be objects and are garbage collected when there are no other references. Map holds strong references to keys and is iterable. WeakMap is not iterable and has no .size property. Use WeakMap for associating metadata with DOM elements without preventing garbage collection. WeakSet has the same relationship to Set. In Playwright, WeakMap is useful for caching element references without memory leaks.

What is NaN? How do you reliably check if a value is NaN?IntermediateP1
Basic Answer

NaN (Not-a-Number) is a global property representing an unrepresentable numeric value. It's the only JavaScript value that is not equal to itself (NaN !== NaN is true). Use Number.isNaN() to reliably check for NaN. The global isNaN() coerces values to numbers first — isNaN('string') returns true, while Number.isNaN('string') returns false.

What are the primitive data types in JavaScript? How do they differ from reference types?BeginnerP2
Basic Answer

Primitive types: string, number, boolean, null, undefined, symbol, bigint. They are immutable and stored by value in the call stack. Reference types: objects, arrays, functions. They are mutable and stored by reference in the heap. When assigning or comparing reference types, the reference (memory address) is used, not the actual content. This is critical in test automation for understanding assertion behavior — two identical objects are not equal unless deeply compared.

Real Interview Answer

"JavaScript has 7 primitive types and reference types. Primitives are stored by value — when you assign let b = a where a is a number, you get a copy. Objects and arrays are stored by reference — assigning copies the reference, not the data. This matters in test automation when using toBe vs toEqual in assertions. toBe checks reference equality, toEqual does deep value comparison."

What are truthy and falsy values in JavaScript? How do they affect conditional logic in tests?BeginnerP2
Basic Answer

Falsy values (6 total): false, 0, '' (empty string), null, undefined, NaN. Everything else is truthy including '0', [] (empty array), {} (empty object), and 'false'. This is essential for writing correct conditional logic in test scripts — for example, checking if an element exists: if (element) will be truthy for any object, even an empty one.

Detailed Answer
// Common pitfalls in test automation const count = 0; if (count) { /* This won't execute — 0 is falsy! */ } const arr: string[] = []; if (arr.length) { // 0 is falsy — correct way to check empty array } // Truthy check for "value exists" — wrong for empty string or 0 if (inputValue) { /* Skipped if value is empty string */ } // Use explicit check instead: if (inputValue !== null && inputValue !== undefined) { }

"Knowing truthy/falsy values prevents subtle bugs. For example, checking if (response.data) on an empty array [] is truthy — the correct check is if (response.data.length > 0). Similarly, checking if (errorCode) fails when errorCode is 0, which might be a valid error code. I always use explicit comparisons in test assertions."

What is currying? How is it useful in test configuration?AdvancedP2
Basic Answer

Currying transforms a function taking multiple arguments into a sequence of functions each taking a single argument. f(a, b, c) → f(a)(b)(c). Useful for creating partially applied test helpers — e.g., const createUser = (role) => (overrides) => baseUserConfig. Enables reusable configuration factories in test frameworks.

How do you remove duplicates from an array? Compare Set vs filter approaches.IntermediateP2
Basic Answer

[...new Set(arr)] is the simplest for primitive values. For objects, use filter with a tracking set: arr.filter((item, idx) => arr.findIndex(i => i.id === item.id) === idx). Set is O(n) and more readable; filter with findIndex is O(n²) but works for objects. For large arrays with objects, use a Map with a unique key.

What is memoization? Implement a memoized function.AdvancedP2
Basic Answer

Memoization caches function results based on arguments to avoid redundant computation. Useful for expensive operations like complex data transformations in test fixtures. The cache is typically a Map or object keyed by the serialized arguments.

function memoize(fn) { const cache = new Map(); return function(...args) { const key = JSON.stringify(args); if (cache.has(key)) return cache.get(key); const result = fn.apply(this, args); cache.set(key, result); return result; }; }

What is the difference between splice() and slice()?BeginnerP2
Basic Answer

slice(start, end) returns a new array with elements from start to (end-1). It does NOT modify the original array. splice(start, deleteCount, ...items) modifies the original array by removing/replacing elements. slice is for reading/copying; splice is for mutation.

How do you flatten a nested array? Implement both recursive and flat() approaches.IntermediateP2
Basic Answer

arr.flat(depth) is the native approach. For unlimited depth: arr.flat(Infinity). A recursive approach uses reduce with concat. In test automation, flattening is useful for processing multi-level API response structures.

What are generators and iterators? When would you use them?AdvancedP2
Basic Answer

Generators (function*) are functions that can be paused and resumed with yield. They return an iterator with .next(). Useful for lazy evaluation of large datasets, infinite sequences, or custom iteration logic in test data generation.

37
How does short-circuit evaluation work with && and ||?BeginnerP2
Basic Answer

&& returns the first falsy operand or the last operand if all are truthy. || returns the first truthy operand or the last operand if all are falsy. Short-circuit means evaluation stops as soon as the result is determined. Used for conditional execution: isLoggedIn && redirect() or default values: const name = user.name || 'Guest'.

How does forEach() differ from map()? When should you avoid forEach()?BeginnerP2
Basic Answer

forEach() executes a function for each element but returns undefined. map() returns a new array with transformed elements. Use forEach for side effects (logging, external calls). Use map for transformations. In test automation, prefer map for data extraction and forEach only when the operation has side effects like element interaction.

For counting: items.reduce((acc, item) => ({...acc, [item]: (acc[item] || 0) + 1}), {}). For flattening: nested.reduce((acc, arr) => acc.concat(arr), []).

What is an IIFE? Why would you use one?IntermediateP2
Basic Answer

An IIFE (Immediately Invoked Function Expression) is a function that runs as soon as it's defined: (function(){ ... })(). Creates a private scope for variables, avoiding global namespace pollution. Used in module patterns before ES6 modules, and still useful for creating isolated scopes in scripts.

What are template literals and tagged templates? Give a Playwright locator example.IntermediateP2
Basic Answer

Template literals use backticks and support interpolation ${expr}. Tagged templates allow processing template literals with a function. Useful for building dynamic locators: page.locator(\`[data-testid="user-\${userId}"]\`).

What is the difference between local storage, session storage, and cookies in browser storage?IntermediateP2
Basic Answer

Local storage persists until explicitly cleared and is shared across tabs. Session storage persists only for the current tab and is cleared when the tab closes. Cookies are sent with every HTTP request (limited to 4KB) and can have expiration dates. In Playwright, page.evaluate(() => localStorage.getItem('token')) accesses storage. Use storageState for preserving auth across tests.

How does the typeof operator work? What is typeof null?BeginnerP2
Basic Answer

typeof returns a string indicating the type. typeof null returns 'object' — a well-known bug from the early days of JavaScript. Use value === null to check for null. Other quirks: typeof NaN returns 'number', typeof [] returns 'object' (use Array.isArray() instead).

How do you implement a simple Promise from scratch?ExpertP2
Basic Answer

A basic Promise implementation needs a state machine (pending/fulfilled/rejected), an executor function, resolve/reject callbacks, and .then() for chaining. The executor runs synchronously. .then() queues callbacks that execute when the state changes (or immediately if already settled). Microtask scheduling requires queueMicrotask for proper async behavior.

45
How does JavaScript handle type coercion? Give examples of implicit vs explicit coercion.IntermediateP2
Basic Answer

Implicit coercion: '5' - 3 → 2 (string to number), '5' + 3 → '53' (number to string), !0 → true. Explicit coercion: Number('5'), String(5), Boolean(0). The + operator favors string concatenation if either operand is a string.

46
What is the difference between for...in and for...of?BeginnerP2
Basic Answer

for...in iterates over enumerable property keys (including inherited ones from the prototype chain). Use for objects. for...of iterates over iterable values (arrays, strings, Maps, Sets). Use for arrays and iterables. for...in on arrays can include prototype properties — avoid it for arrays.

47
How does Object.create() work? Compare with constructor functions and class syntax.AdvancedP2
Basic Answer

Object.create(proto) creates a new object with the specified prototype. This is pure prototypal inheritance without constructors. Constructor functions (new Fn()) and classes (class) are syntactic sugar over prototypes. Object.create(null) creates an object with no prototype — useful for pure dictionaries without inherited properties.

What is the difference between Array.isArray() and instanceof Array?IntermediateP2
Basic Answer

Array.isArray() is the reliable way to check for arrays. instanceof Array can fail across iframe/realm boundaries where the Array constructor differs. Array.isArray() works correctly even with arrays from different execution contexts.

How does Intl work for localization testing?AdvancedP3

The Intl API provides locale-aware formatting for dates, numbers, currencies, and collation. Useful in localization testing to verify that application output matches expected locale formats. new Intl.DateTimeFormat('de-DE').format(date) vs 'en-US'.

What is the Reflect API? How does it relate to Proxy?ExpertP3
Basic Answer

Reflect is a global object with methods that correspond to Proxy traps (Reflect.get(), Reflect.set(), etc.). It provides the default behavior for Proxy traps and is the recommended way to forward operations to the target object within traps.

obj.hasOwnProperty('key') or Object.hasOwn(obj, 'key') checks only own properties. 'key' in obj checks both own and inherited properties. obj.key !== undefined is not reliable because the property could exist but have value undefined.

What is the difference between requestAnimationFrame and setTimeout?AdvancedP3
Basic Answer

requestAnimationFrame schedules a function before the next browser repaint (60fps sync). setTimeout schedules after a minimum delay regardless of rendering. requestAnimationFrame pauses when the tab is inactive (saves resources). Useful for visual testing and animation testing.

What are Web Workers and how do they relate to the event loop?ExpertP3
Basic Answer

Web Workers run JavaScript in a background thread with their own event loop and call stack. They communicate with the main thread via postMessage. They have no DOM access. Useful for offloading heavy computation in test data generation without blocking the UI.

📋 JavaScript — One-Page Revision Cheat Sheet

Scope & Hoisting

var → function scope, hoisted with undefined

let/const → block scope, hoisted uninitialized (TDZ)

Function declarations → fully hoisted

Closures

Function + its lexical scope preserved

Used for: private state, module pattern, retry logic

Risk: memory leaks if capturing large objects

Event Loop Order

1. Sync code (call stack)

2. Microtasks (Promise.then, queueMicrotask)

3. One macrotask (setTimeout, setInterval)

4. Repeat

this Binding (Priority)

new > explicit (call/apply/bind) > implicit (obj.method) > default (global)

Arrow functions: lexical, cannot be overridden

Promise Static Methods

Promise.all → fail-fast on first rejection

Promise.allSettled → wait for all, never rejects

Promise.race → settle on first settled

Promise.any → first fulfilled (rejects if all reject)

Object Methods

freeze → no add/delete/modify

seal → no add/delete, can modify

preventExtensions → no add

All are shallow

Array Methods

map → transform (returns new array)

filter → subset (returns new array)

reduce → accumulate (any type)

find → first match (returns element)

some/every → boolean check

Clone Deep vs Shallow

Shallow: spread (...obj), Object.assign()

Deep: structuredClone(), JSON.parse/stringify

Watch out: functions, Dates, Maps, Sets, circular refs
