P1 — Frequently Asked Questions

Must-know SQL queries and database concepts for SDET interviews.

1
Write a query to find the second highest salary from an employee table (multiple approaches).IntermediateP1
Basic Answer

Three main approaches: (1) Subquery with MAX: SELECT MAX(salary) FROM employees WHERE salary < (SELECT MAX(salary) FROM employees). (2) LIMIT/OFFSET: SELECT DISTINCT salary FROM employees ORDER BY salary DESC LIMIT 1 OFFSET 1. (3) Window function: SELECT DISTINCT salary FROM (SELECT salary, DENSE_RANK() OVER (ORDER BY salary DESC) as rnk FROM employees) WHERE rnk = 2.

Tip Interviewers value showing multiple approaches. The window function approach is the most scalable for Nth highest salary.
2
Write a query to find and delete duplicate records while keeping one occurrence.IntermediateP1
Basic Answer

Finding duplicates: SELECT email, COUNT(*) FROM users GROUP BY email HAVING COUNT(*) > 1. Deleting duplicates (keeping the first): DELETE FROM users WHERE id NOT IN (SELECT MIN(id) FROM users GROUP BY email). Or using ROW_NUMBER(): DELETE FROM (SELECT *, ROW_NUMBER() OVER (PARTITION BY email ORDER BY id) as rn FROM users) WHERE rn > 1.

3
Explain INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN, and SELF JOIN with examples.IntermediateP1
Basic Answer

INNER JOIN: returns only matching rows from both tables. LEFT JOIN: all rows from left table, matching rows from right (NULLs where no match). RIGHT JOIN: opposite of LEFT. FULL OUTER JOIN: all rows from both tables (NULLs where no match). SELF JOIN: joining a table to itself (aliased). Most commonly asked: INNER JOIN and LEFT JOIN in SDET interviews.

-- SELF JOIN: Employees earning more than their manager SELECT e1.name AS employee, e1.salary, e2.name AS manager FROM employees e1 INNER JOIN employees e2 ON e1.manager_id = e2.id WHERE e1.salary > e2.salary;
4
What is the difference between WHERE and HAVING? When would you use each?BeginnerP1
Basic Answer

WHERE filters rows before aggregation (GROUP BY). HAVING filters groups after aggregation. WHERE cannot use aggregate functions; HAVING can. Execution order: WHERE → GROUP BY → HAVING → ORDER BY. Example: SELECT department, AVG(salary) FROM employees WHERE salary IS NOT NULL GROUP BY department HAVING AVG(salary) > 50000.

5
Write a query using window functions: ROW_NUMBER(), RANK(), DENSE_RANK(), and NTILE().AdvancedP1
Basic Answer

ROW_NUMBER() assigns unique sequential numbers (ties get different numbers). RANK() assigns same rank to ties but skips subsequent numbers. DENSE_RANK() assigns same rank to ties without skipping. NTILE(n) divides rows into n equal buckets. All used with OVER (PARTITION BY col ORDER BY col).

-- Top 3 salaries per department SELECT department, name, salary, DENSE_RANK() OVER (PARTITION BY department ORDER BY salary DESC) as rnk FROM employees QUALIFY rnk <= 3; -- QUALIFY filters window function results (Snowflake/Presto) -- Alternative: Use subquery: SELECT * FROM (...) WHERE rnk <= 3
6
Write a query using LEAD() and LAG() to find month-over-month growth.AdvancedP1
Basic Answer

LAG() accesses a previous row's value; LEAD() accesses a next row's value. Both require OVER (ORDER BY col). For month-over-month growth: SELECT month, revenue, LAG(revenue) OVER (ORDER BY month) as prev_month_revenue, ROUND((revenue - LAG(revenue) OVER (ORDER BY month)) / LAG(revenue) OVER (ORDER BY month) * 100, 2) as growth_pct FROM monthly_revenue.

7
Write a correlated subquery. How does it differ from a regular subquery?AdvancedP1
Basic Answer

A correlated subquery references columns from the outer query and executes once for each row in the outer query. A regular (non-correlated) subquery executes once and its result is reused. Correlated subqueries are less efficient but more expressive. Example: SELECT e1.name, e1.salary FROM employees e1 WHERE e1.salary > (SELECT AVG(e2.salary) FROM employees e2 WHERE e2.department = e1.department).

8
Find customers with no orders using multiple approaches.IntermediateP1
Basic Answer

LEFT JOIN: SELECT c.* FROM customers c LEFT JOIN orders o ON c.id = o.customer_id WHERE o.id IS NULL. NOT EXISTS: SELECT * FROM customers c WHERE NOT EXISTS (SELECT 1 FROM orders o WHERE o.customer_id = c.id). NOT IN: SELECT * FROM customers WHERE id NOT IN (SELECT customer_id FROM orders) (cautious with NULLs). NOT EXISTS is usually most performant.

9
Write a recursive CTE (Common Table Expression) for hierarchical data like an org chart.ExpertP1
Basic Answer

A recursive CTE has an anchor member (initial query) and a UNION ALL with the recursive member (references itself). Example: WITH RECURSIVE org_chart AS (SELECT id, name, manager_id, 1 as level FROM employees WHERE manager_id IS NULL UNION ALL SELECT e.id, e.name, e.manager_id, oc.level + 1 FROM employees e INNER JOIN org_chart oc ON e.manager_id = oc.id) SELECT * FROM org_chart ORDER BY level, name.

10
What is the difference between clustered and non-clustered indexes?AdvancedP1
Basic Answer

A clustered index determines the physical order of data in a table — each table can have only one. A non-clustered index is a separate structure storing the index key and a pointer to the data row. Clustered indexes are faster for range queries and sorting. Non-clustered indexes are faster for lookups that don't need all columns. An index seek is more efficient than an index scan. Use clustered indexes for primary keys on OLTP tables.

11
Explain the ACID properties of database transactions.IntermediateP1
Basic Answer

Atomicity: All operations in a transaction complete, or none do (all-or-nothing). Consistency: Transactions bring the database from one valid state to another, preserving constraints. Isolation: Concurrent transactions don't interfere with each other (controlled by isolation levels). Durability: Committed transactions persist even after system failure. In test automation, ACID ensures test data is either fully set up or fully rolled back.

12
What is the difference between DELETE, TRUNCATE, and DROP?BeginnerP1
Basic Answer

DELETE: DML command, removes rows one by one (slower), can use WHERE, triggers fire, can be rolled back. TRUNCATE: DDL command, removes all rows instantly by deallocating pages, cannot use WHERE, fewer triggers, auto-increment resets. DROP: DDL command, removes the entire table structure and data. In test automation, TRUNCATE is preferred for fast cleanup between test suites.

13
Explain primary key, foreign key, unique key, and composite key differences.BeginnerP1
Basic Answer

Primary Key: Unique identifier for each row, NOT NULL, one per table. Foreign Key: References a primary key in another table, enforces referential integrity. Unique Key: Ensures all values are unique, allows one NULL. Composite Key: Primary key using multiple columns. Foreign keys can cascade updates/deletes to maintain consistency across related tables.

14
How do you validate data between an API response and a database in test automation?AdvancedP1
Basic Answer

Query the database to get the expected data, call the API to get the actual response, then compare. Use parameterized queries to avoid SQL injection. Handle data type differences (SQL vs JSON). For timestamps, use absolute comparisons with tolerances. For large datasets, compare checksums or counts rather than full records. The validation should be part of the test: const dbData = await db.query('SELECT * FROM users WHERE id = ?', [userId]); const apiData = await api.getUser(userId); expect(apiData).toMatchObject(dbData).

15
What is database normalization? Explain 1NF, 2NF, and 3NF with examples.IntermediateP1
Basic Answer

1NF: Each column has atomic (indivisible) values, no repeating groups. 2NF: In 1NF plus all non-key columns fully depend on the entire primary key (no partial dependencies). 3NF: In 2NF plus no transitive dependencies (non-key columns don't depend on other non-key columns). Normalization reduces data redundancy, but denormalization can improve read performance for analytics.

P2 — Commonly Asked Questions

Beginner-level SQL queries frequently asked in technical screens.

16
Write a basic SELECT query with a WHERE clause. How do you filter records?BeginnerP2
Basic Answer

SELECT * FROM employees WHERE status = 'Active'. The WHERE clause filters rows based on conditions using comparison operators (=, <>, >, <, >=, <=). For string comparisons, most databases are case-insensitive by default. Use AND/OR to combine multiple conditions. In test automation, SELECT with WHERE is used to verify specific records created by a test.

SELECT id, name, email, status FROM users WHERE status = 'ACTIVE' AND created_at >= '2024-01-01';
17
How does ORDER BY work with ASC and DESC? How do LIMIT/OFFSET work for pagination?BeginnerP2
Basic Answer

ORDER BY col ASC sorts ascending (default), DESC sorts descending. Can sort by multiple columns: ORDER BY department ASC, salary DESC. LIMIT n returns only n rows. OFFSET m skips m rows. LIMIT 10 OFFSET 20 returns rows 21-30 (for pagination). In test automation, ORDER BY is essential for verifying sorted results, and LIMIT/OFFSET helps sample data from large tables.

18
Write a query using COUNT, SUM, AVG, MIN, MAX aggregate functions.BeginnerP2
Basic Answer

COUNT(*) counts rows, COUNT(col) counts non-NULL values. SUM() adds numeric values. AVG() calculates average. MIN() and MAX() find extreme values. Aggregate functions ignore NULLs except COUNT(*). Use for data validation in test reports.

SELECT COUNT(*) AS total_orders, SUM(amount) AS total_revenue, AVG(amount) AS avg_order_value, MIN(amount) AS smallest_order, MAX(amount) AS largest_order FROM orders WHERE created_at >= '2024-01-01';
19
How does GROUP BY work? Write a query to count employees per department.BeginnerP2
Basic Answer

GROUP BY groups rows sharing a value, then applies aggregate functions per group. Every column in SELECT must either be in GROUP BY or wrapped in an aggregate function. Use HAVING to filter groups (like WHERE for groups).

SELECT department, COUNT(*) AS employee_count FROM employees GROUP BY department HAVING COUNT(*) > 5 -- Only departments with more than 5 employees ORDER BY employee_count DESC;
20
How do the BETWEEN, IN, and LIKE operators work? Give SDET examples.BeginnerP2
Basic Answer

BETWEEN is inclusive: salary BETWEEN 50000 AND 100000. IN checks a set: status IN ('Active', 'Pending'). LIKE does pattern matching: % matches any sequence, _ matches a single character. LIKE 'john%' finds names starting with 'john'. In test automation, LIKE is useful for validation of partial text matches in database verification.

21
How does IS NULL work? How should you handle NULL values in queries?BeginnerP2
Basic Answer

Use IS NULL and IS NOT NULL to check for NULL. Never use = NULL since NULL is not equal to anything (even itself). NULLs are ignored in aggregate functions (COUNT(col) skips NULLs). NULLs sort first or last depending on the database. Understanding NULL behavior is critical for test assertions against database records.

-- Check for NULL values SELECT * FROM users WHERE email IS NULL; -- Correct! -- SELECT * FROM users WHERE email = NULL; -- WRONG! Returns nothing -- Coalesce NULL to a default value SELECT name, COALESCE(phone, 'No Phone') AS contact FROM users;
22
What is the DISTINCT keyword? Write a query to find unique values in a column.BeginnerP2
Basic Answer

SELECT DISTINCT column returns unique values, removing duplicates. SELECT DISTINCT city FROM customers gives all unique cities. Multiple columns: SELECT DISTINCT city, state FROM customers returns unique combinations. In test automation, DISTINCT is used for validating lookup values and data integrity.

23
How do string functions like CONCAT, UPPER, LOWER, and LENGTH work?BeginnerP2
Basic Answer

CONCAT(str1, str2) joins strings (or use || operator). UPPER(str) and LOWER(str) change case — useful for case-insensitive comparisons. LENGTH(str) returns character count. Also useful: TRIM() removes whitespace, SUBSTRING(str, start, length) extracts parts. In test automation, these functions normalize data for comparison.

SELECT CONCAT(first_name, ' ', last_name) AS full_name, LOWER(email) AS email_normalized, LENGTH(phone) AS phone_digits FROM users;
24
How does the CASE statement work for conditional logic in SQL queries?BeginnerP2
Basic Answer

CASE provides if-then-else logic in SQL. Two forms: simple (CASE col WHEN val1 THEN result1 END) and searched (CASE WHEN condition THEN result END). Useful in test automation for categorizing data, applying test statuses, or transforming values for easier comparison.

SELECT name, salary, CASE WHEN salary > 100000 THEN 'High' WHEN salary > 50000 THEN 'Medium' ELSE 'Low' END AS salary_level FROM employees;
25
Write a basic INNER JOIN to fetch related data from two tables.BeginnerP2
Basic Answer

INNER JOIN returns only rows with matching keys in both tables. Use ON to specify the join condition. Table aliases improve readability. This is the most common join in test automation for verifying related records across tables.

-- Fetch orders with customer details SELECT o.id AS order_id, c.name AS customer_name, o.total_amount, o.order_date FROM orders o INNER JOIN customers c ON o.customer_id = c.id WHERE o.status = 'COMPLETED' ORDER BY o.order_date DESC;
26
Write a query to find the Nth highest salary without using LIMIT/OFFSET.IntermediateP2
Basic Answer

Use a correlated subquery: SELECT DISTINCT salary FROM employees e1 WHERE N-1 = (SELECT COUNT(DISTINCT salary) FROM employees e2 WHERE e2.salary > e1.salary). Or use window function: SELECT salary FROM (SELECT salary, DENSE_RANK() OVER (ORDER BY salary DESC) as rnk FROM employees) WHERE rnk = N LIMIT 1.

27
What is the difference between UNION and UNION ALL?BeginnerP2
Basic Answer

UNION removes duplicate rows and sorts the result. UNION ALL keeps all rows including duplicates and does not sort. UNION ALL is faster because it avoids the deduplication step. Use UNION when you need distinct results across multiple SELECT statements.

28
What is an execution plan? How do you use EXPLAIN to analyze query performance?AdvancedP2
Basic Answer

An execution plan shows how the database engine processes a query — which indexes are used, join types, scan methods. Use EXPLAIN SELECT ... to view it. Look for: table scans (bad → missing index), index seeks (good), join types (Nested Loops vs Hash Match vs Merge Join), estimated rows vs actual rows (large deviations suggest outdated statistics).

29
How do you handle NULL values in sorting? How do you sort NULLs last?IntermediateP2
Basic Answer

NULL handling in sorting is database-specific. In PostgreSQL: ORDER BY column NULLS LAST. In MySQL: ORDER BY ISNULL(column), column. In SQL Server: NULLs are lowest by default. Use COALESCE(column, replacement) to replace NULLs before sorting if consistent ordering is needed.

30
What are the isolation levels in SQL? Explain dirty read, non-repeatable read, and phantom read.AdvancedP2
Basic Answer

Read Uncommitted: can see uncommitted data (dirty read). Read Committed: only sees committed data (default in most databases) — prevents dirty reads. Repeatable Read: same row read twice returns same data — prevents non-repeatable reads. Serializable: highest isolation, prevents phantoms (rows appearing/disappearing) by locking ranges. Higher isolation = better consistency, lower concurrency.

P3 — Valuable Context

31
Explain the CAP theorem. How does it apply to NoSQL databases?ExpertP3
Basic Answer

CAP theorem states a distributed database can only guarantee two of three: Consistency, Availability, Partition Tolerance. Most NoSQL databases choose AP (Cassandra) or CP (MongoDB, HBase). Relational databases typically focus on CA within a single node. Understanding CAP helps determine which database to use based on requirements: banking (CP) vs social media (AP).

32
How do you optimize a slow query? Walk through your investigation process.AdvancedP3
Basic Answer

Steps: (1) Run EXPLAIN to see the execution plan. (2) Check for full table scans — missing indexes. (3) Review join types — are they using appropriate indexes? (4) Check WHERE clause — are indexed columns being used with functions or type casting? (5) Review the number of rows being processed vs needed. (6) Check for N+1 queries (multiple queries in loops). (7) Consider query restructuring, denormalization, or caching.

33
How do you seed and clean up test data for parallel database testing?AdvancedP3
Basic Answer

Approaches: (1) Use database transactions that roll back after each test (fastest, but doesn't catch all state issues). (2) Create unique data per test using UUIDs or timestamps in test data. (3) Use dedicated test databases created per worker/container. (4) Use TRUNCATE between test suites. (5) For read-only tests, use a shared snapshot. The key is data isolation — tests should not share mutable state.

34
How do you test ETL pipelines? What validations are critical?ExpertP3
Basic Answer

Critical ETL validations: (1) Row count matching between source and target. (2) Data type and format transformations are correct. (3) Null handling — are NULLs preserved or replaced? (4) Duplicate handling — are duplicates removed as expected? (5) Incremental vs full load — does incremental load produce the correct delta? (6) Reject handling — are failed rows logged? (7) Performance — does the pipeline meet SLA?

SQL — One-Page Revision Cheat Sheet

Join Reference

INNER: only matches. LEFT: all left + matches. RIGHT: all right + matches. FULL: all rows. SELF: table to itself.

Window Functions

ROW_NUMBER (unique), RANK (with gaps), DENSE_RANK (no gaps), NTILE (buckets), LAG/LEAD (previous/next).

Execution Order

FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT

Index Types

Clustered: physical order, 1 per table. Non-clustered: separate structure, many per table.

SQL vs NoSQL

SQL: ACID, structured, joins, vertical scaling. NoSQL: flexible schema, horizontal scaling, CAP tradeoffs.

DDL vs DML vs DCL

DDL: CREATE, ALTER, DROP, TRUNCATE. DML: SELECT, INSERT, UPDATE, DELETE. DCL: GRANT, REVOKE.
